What is seo and what are the methods in seo? 
**SEO (Search Engine Optimization)** is the process of optimizing a website to improve its visibility in search engine results, ultimately increasing organic (non-paid) traffic.

**Key SEO Methods:**

1. **Keyword Research:** Identifying relevant keywords and phrases to target in your content.
2. **On-Page SEO:** Optimizing individual pages by improving content quality, using headings, meta tags, and images.
3. **Technical SEO:** Ensuring the website is crawlable, has a good site structure, fast loading times, and is mobile-friendly.
4. **Off-Page SEO:** Building backlinks from reputable sites to improve domain authority.
5. **Content Marketing:** Creating high-quality, valuable content that addresses user needs and encourages sharing.

These methods collectively enhance a website's ranking in search engines, leading to increased visibility and traffic.

what are meta tags? 
Meta tags are HTML elements used to provide structured metadata about a web page. They are placed in the  section of the HTML document and help search engines understand the content of the page. Common meta tags include:

      meta name="description": Summarizes the page content for search engines and is often displayed in search results.
      meta name="keywords": Lists relevant keywords for the page (less commonly used today).
      meta name="viewport": Controls the layout on mobile browsers.
      meta charset="UTF-8": Specifies the character encoding for the document.
      These tags play a crucial role in SEO and responsive design.
What is iframes? 
iFrames (inline frames) are HTML elements used to embed another document within the current HTML document. They allow you to display content from another source, such as a webpage or video, without leaving the main page. iFrames are commonly used for ads, maps, and other interactive content.

what are the features of html 5? 
HTML5 includes several key features:

      1. **New Semantic Elements**: Introduces elements like for better document structure.
      
      2. **Multimedia Support**: Native support for audio and video elements without the need for plugins.
      
      3. **Canvas Element**: Allows for dynamic, scriptable rendering of 2D shapes and bitmap images using the tag.
      
      4. **Form Enhancements**: New input types (e.g., `email`, `date`, `number`) and attributes (e.g., `placeholder`, `autofocus`) for improved form functionality.

      8. **Responsive Design**: Media queries and flexible layouts support responsive web design.
      
      These features enhance web applications' capabilities, making them more interactive and efficient.
what is git? 
Git is a distributed version control system that helps track changes in source code during software development. It allows multiple developers to collaborate on projects, manage versions, and revert to previous states of the code easily. Git maintains a complete history of changes, making it essential for managing large codebases efficiently.
differnce between git vs github 
Git: A distributed version control system that allows developers to track changes in their code and collaborate on projects. It operates locally on a user's machine.

      GitHub: A web-based platform that hosts Git repositories, providing a collaborative environment for developers to share code, manage projects, and contribute through features like pull requests, issues, and wikis.
      
      In short, Git is the tool for version control, while GitHub is a platform for hosting and collaborating on Git repositories.

what is the flow of git??? 
The Git workflow typically follows these steps:

      Clone: Copy a remote repository to your local machine.
      Branch: Create or switch to a separate branch for new work.
      Modify: Make changes to files and stage them.
      Commit: Save your changes locally with a message describing them.
      Push: Send your local commits to the remote repository.
      Pull: Fetch and merge changes from the remote repository.
      Merge/Rebase: Combine your branch with another branch (like main or master).
      Resolve Conflicts: Manually fix any merge conflicts if needed.
      This workflow ensures smooth collaboration and version control.
